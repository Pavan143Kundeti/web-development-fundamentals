# Web Development Introduction

## What is Web Development?

Web development is the process of creating websites. It involves building web pages that users can view and interact with through web browsers.

## Key Terms You Must Know

### 1. Website
- A **website** is a collection of web pages
- Multiple web pages linked together form a complete website
- Each page serves a specific purpose or contains specific content

### 2. Web Page
- A **web page** is an HTML document
- Contains HTML elements that define the structure and content
- Users see the formatted version in their browser

### 3. Web Server
- **Web server** is software that stores and delivers web pages
- When you request a website, the web server sends the web page to your browser
- Examples: Apache, Nginx, IIS

### 4. Web Client
- **Web client** is software that requests web pages from servers
- Most common web clients are web browsers
- Examples: Chrome, Firefox, Safari, Edge

## How Websites Work

### The Request-Response Process

1. **User Action**: You type a website name (like ndtv.com) in your browser
2. **DNS Lookup**: Browser finds the IP address of the website
3. **Request Sent**: Browser sends HTTP request to the web server
4. **Server Response**: Web server sends back the web page
5. **Display**: Browser displays the formatted web page

### Domain Names and IP Addresses

| Component | Purpose | Example |
|-----------|---------|---------|
| Domain Name | Human-readable website address | ndtv.com |
| IP Address | Unique number identifying server location | 192.168.1.1 |
| DNS Server | Converts domain names to IP addresses | Automatic process |

## Website vs Web Application

### Website (Static)
- **Purpose**: Display information only
- **User Interaction**: Limited to viewing content
- **Data Flow**: One-way (server to client)
- **Examples**: News sites, blogs, company information pages

### Web Application (Dynamic)
- **Purpose**: Interactive functionality
- **User Interaction**: Users can input and modify data
- **Data Flow**: Two-way (client to server and back)
- **Examples**: Gmail, Facebook, Amazon, online banking

## Static vs Dynamic Web Pages

### Static Web Pages
- **Content**: Never changes
- **Same for Everyone**: All users see identical content
- **Examples**: About us pages, contact information
- **Technology**: Pure HTML, CSS

### Dynamic Web Pages
- **Content**: Changes based on various factors
- **Personalized**: Different content for different users
- **Factors Affecting Content**:
  - User location
  - Time of day
  - User preferences
  - Login status
- **Examples**: 
  - Live cricket scores
  - Weather reports
  - Social media feeds
  - Shopping cart contents

## Web Development Process

### Two Main Stages

#### 1. Frontend Development (Client-Side)
- **What Users See**: Visual design and user interface
- **Technologies Used**:
  - HTML (structure)
  - CSS (styling)
  - JavaScript (interactivity)
- **Responsibility**: How the website looks and behaves in the browser

#### 2. Backend Development (Server-Side)
- **What Users Don't See**: Server logic and database operations
- **Technologies Available**:
  - PHP
  - Java
  - Python
  - Node.js
  - Ruby
- **Responsibility**: Processing requests, managing data, business logic

## HTTP Protocol

### What is HTTP?
- **HTTP** = HyperText Transfer Protocol
- Set of rules for communication between web clients and servers
- Defines how requests and responses should be formatted

### HTTP Request and Response
- **Request**: Browser asks server for a web page
- **Response**: Server sends the requested web page back
- Both follow specific formats defined by HTTP protocol

## Learning Path Recommendation

### Step 1: Master Frontend Basics
1. **HTML** - Structure and content
2. **CSS** - Styling and layout  
3. **JavaScript** - Interactivity and behavior

### Step 2: Learn Frontend Frameworks (Optional)
- **CSS Framework**: Bootstrap (faster styling)
- **JavaScript Frameworks**: React, Vue, Angular (advanced functionality)

### Step 3: Choose Backend Technology
- Pick one language based on your interest
- Master the basics before moving to frameworks

### Step 4: Learn Backend Frameworks
- **Python**: Django, Flask
- **Java**: Spring
- **PHP**: Laravel
- **JavaScript**: Node.js with Express

### Step 5: Deployment
- **Purpose**: Make your website accessible to users worldwide
- **Process**: Upload your website to a web server
- **Hosting Services**: AWS, Heroku, Netlify, GitHub Pages

## Key Points to Remember

- Start with HTML, CSS, and JavaScript before moving to backend
- Choose one backend language and master it completely
- Frameworks save development time but learn basics first
- Deployment is essential to make your website publicly accessible
- Practice building complete projects to understand the full process
